## Automatic Branch Detection

When you say "push", the system automatically detects which branch to use:

### Frontend Changes
- **Files:** Any file in `frontend/` directory
- **Branch:** `sow-latest`
- **Command:** `git push origin sow-latest`

### Backend Changes
- **Files:** Any file in `backend/` directory  
- **Branch:** `backend-service`
- **Command:** `git push origin backend-service`

### Mixed Changes
- If both frontend and backend files are changed, you must push them separately:
  1. Commit frontend changes ‚Üí push to `sow-latest`
  2. Commit backend changes ‚Üí push to `backend-service`

## Workflow

1. **Always work in:** `/root/the11-dev-clean`
2. **Make changes** to frontend or backend files
3. **Stage changes:** `git add <files>`
4. **Commit:** `git commit -m "message"`
5. **Push:** 
   - Frontend ‚Üí `git push origin sow-latest`
   - Backend ‚Üí `git push origin backend-service`
